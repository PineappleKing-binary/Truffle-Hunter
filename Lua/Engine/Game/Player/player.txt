this is the first basic idea.
we need 2 scripts for this
first is playerbox.lua
this script will make it so that way the player always has a hitbox
the x and y size of the player hitbox will be two vars so we can fine tune it to be perfect, no object that also contains a hitbox can go through or inside of the player hitbox. 
the basic movement is wasd. s attacks (no kicking sprites yet), a and d move player left and right as you'd expect, and w jumps. we also have vars for the left and right movement speed, the jump force and more.
playerbox also contains the gravity formula. the player object can never rotate and follows the gravity formula of 
velocity = velocity + acceleration * deltaTime
position = position + velocity * deltaTime

we can fine tune velocity, and acceleration, the delta time would be calculated from jump force.
the final script is anims.lua
this controls the player sprites
when the player is not moving left or right, it always uses IMG/JUBBA/F1W.png as an idle pose. 
when the player jumps, it will still use F1W, but when it is falling it uses walking anim. 

heres how walking anim works :
it is 15 frames per second and it goes 
F1W
F2W
F3W
F4W
F5W
F6W
F7W
F8W

then loops back to f1w. this anim is played when moving left or right or falling down after a jump
by default the sprites face right, so when the player is moving right (d movement), it doesnt change sprites, but when moving left (a movement)'
the sprites get flipped horizontally. and get flipped back to normal when moving right again.
